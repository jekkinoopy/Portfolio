# 網站架構分類草案 v1

> 組織原則：Nav 列＝「創作媒介／領域」的清單，可以持續加新項目；不掛 Nav 的頁面靠連結分享，跟「這是不是作品」無關——開放範圍是「有沒有掛 Nav」決定的，不是內容類型決定的。
> 完整視覺版（分層圖）：對話中產生的 Artifact，本檔為文字存檔版本。

## Layer A — 頂層 Nav（創作媒介／領域，可持續擴充）

這層可以一直長，新增一個就是新增一個「領域」，跟現有的 Graphic、Photo 同一種待遇。

**已開放**
- Graphic（+6 項籌備子分類：名片/卡片/票券/DM、網站視覺 Banner、社群推播圖、大圖輸出、書刊、商品長條圖文）
- Illustration
- UI/UX（9 個案例）
- Web

**籌備中**
- About
- Photo（3 項子頁已建檔：landscape / pets / portrait）

**提案中（同一層，未來可加）**
- 歌唱／演出
- 舞蹈
- 手工藝
- 自製遊戲

### 新增一個 Nav 領域，照現有模式走三步

1. 建 `領域名/index.html`，套用 `page-inner`，先不管美不美。
2. Nav 加一個灰字項目＋`downDisabled`，hover 顯示「作品建置中」——跟 About、Photo 現在的待遇一樣。
3. 東西準備好了再拿掉 `downDisabled`，同時更新 `z_misc/site-guide/` 跟 `z_notes/02_專案檔案結構.md`。

---

## Layer B — z_misc（都是可能公開的內容，只是還沒掛 Nav）

**這層不分「工具」跟「個人書寫」——全部都是你會做出來、也可能對外的東西，差別只在有沒有連結給人。** 哪天想開放，直接在 Nav 加一項或把連結發出去即可，機制跟 Layer A 相通。

- `z_misc/site-guide/` — 站主自用地圖（目前唯一比較確定「不會公開」的一個，因為它本來就是給你自己核對用的工具）
- `z_misc/price/`（+ `price2.html` 另一版排版） — 服務報價
- `z_misc/resume/`（+ `r.html` 精簡版） — 履歷
- `z_misc/notes/` — 閱讀札記／隨筆：`avoiding-mistakes` · `making-your-choice-right` · `brain-press-conference` · `care-needs-order`
- `z_misc/yumi/` — 獨立視覺專題，自有樣式，不套用站版型

---

## Layer C — 純內部（真的不會變成網站內容）

這層才是跟「網站內容」完全無關的東西：待辦清單，跟你交代給 Claude 的規範/指令。

- `z_notes/00_TOMORROW_TODO.md` — 待辦
- `z_notes/01_GIT_兩地協作筆記.md`、`02_專案檔案結構.md`、`03_Photo照片分類建議.md`、`04_Illustration插畫分類建議.md` — 給自己/給 Claude 的專案筆記
- `developer-rules/`、`CLAUDE.md` — 給 Claude 的工作規範

**命名衝突提醒**：`z_notes/`（純內部）跟 `z_misc/notes/`（Layer B，可能公開的札記）名字太像，容易搞混、找不到東西。建議之後把內部筆記改名，例如 `z_dev-notes/`，跟內容用的 `notes/` 徹底分開。
