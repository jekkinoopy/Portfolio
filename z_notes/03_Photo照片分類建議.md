# Photo 照片分類建議

> 站主拍攝重心：**景點／風景為主** → **貓狗** → **人物少數**。  
> 現站 `photo/` 仍是「作品集示範用」四分法（人像、商品、活動、風景），與你實際內容不完全對齊；本筆記先定**怎麼整理檔案**，之後改版頁面再照此執行。  
> 相關：`photo/index.html` · `z_assets/images/photo/` · 全站導覽 `z_misc/site-guide/`

---

## 一、原則（先記這三句）

1. **依「畫面主角」分，不依器材或地點名稱資料夾亂塞。**  
   例：在南雅奇石拍貓 → 若貓是主角進「貓狗」；若只是風景裡的小點、景點才是重點 → 進「景點」。
2. **一張只歸一類**（整理／上站用）；同一次出遊可拆成多個子資料夾，但單檔不要重複複製多份。
3. **網站展示順序跟你的比例一致**：景點最大、貓狗其次、人物精選少量即可；不必為了「四欄對稱」硬湊商品／活動。

---

## 二、建議對外三分類（訪客看到的標題）

| 優先 | 建議名稱 | 放什麼 | 約佔作品比例（你） |
|------|----------|--------|-------------------|
| 1 | **景點／風景** | 旅遊地、海岸山岳、地質（南雅奇石等）、城市地標、建築外觀、空景、夕陽雲海；**無明顯人物或寵物當主角**的紀錄 | 多數（約 60–75%） |
| 2 | **貓狗／寵物** | 貓、狗為畫面主角；居家、戶外、與景點同框但**眼神／構圖明顯以寵物為主** | 次之（約 20–30%） |
| 3 | **人物** | 朋友、街拍人像、活動中「人」是主角；**精選少張**即可，不必獨立撐一整區 | 少數（約 5–15%） |

**暫時不建議當主分類的（除非你之後真的接案）：**

| 現站名稱 | 說明 |
|----------|------|
| 商品／靜物 | 商業產品、食物平拍；你目前非主軸 → 先**不開區**或整區隱藏 |
| 活動紀錄 | 婚禮、演唱會、市集等多人活動；若沒有這類委託 → 併入「人物」精選，或單獨做一個「生活隨拍」小頁即可 |

## 三、本機資料夾怎麼放（整理用）

在 `z_assets/images/photo/` 底下用**主題資料夾**（上站前可先這樣理；不必一次搬完舊檔）：

```text
z_assets/images/photo/
├── places/              # 景點／風景（主力）
│   ├── 210908_南雅奇石/
│   ├── 2024_九份/
│   └── …
├── pets/                # 貓狗
│   ├── 貓_名字或年份/
│   └── 狗_…/
├── people/              # 人物（少數）
│   └── …
├── _archive/            # 還沒決定、或暫不上站
└── _export/             # 已裁切／壓縮要上網的 JPG（可選）
```

**子資料夾命名建議：** `YYYYMMDD_地點或主題`（與你現有 `210908_攝影_南雅奇石` 習慣相容）。

**原始檔：** `.NEF` 等大檔可放同資料夾或本機另存；**上 repo 只放匯出 JPG**（與 UI/UX 大檔規則相同，避免 push 被擋）。

## 四、單張歸類：快速判斷

```text
畫面第一眼看到的是誰？
├─ 風景／建築／地標／空景 ──────────────→ places（景點）
├─ 貓或狗（臉部／身體佔畫面明顯） ───────→ pets（貓狗）
├─ 人的臉或全身是視覺焦點 ─────────────→ people（人物）
└─ 都模糊／測試曝光／重複張 ───────────→ _archive 或刪除
```

**常見灰色地帶：**

| 狀況 | 建議 |
|------|------|
| 景點前的人很小、當比例尺 | **景點** |
| 景點很美但貓坐在前景正中央 | **貓狗**（若上「景點頁」會讓系列風格混亂） |
| 朋友旅行合照、人都清楚 | **人物**；若只做一兩張精選可放人物區，其餘 `_archive` |
| 街景＋路人 | 無主角路人 → **景點**；有故事感街拍 → **人物** |
| 食物／桌面小物 | 非你主軸 → 不進三大類，或將來「生活」小頁再說 |

## 五、過渡期：對應現站四個 HTML（還沒改版前）

現有四頁只是**版型不同**，可先「借殼上架」：

| 你的分類 | 暫用現有頁面 | 版型特點 |
|----------|--------------|----------|
| 景點／風景 | `landscape.html` | 瀑布流、完整比例（最適合橫直混雜的風景） |
| 貓狗 | `product.html` 或 `event.html` | 馬賽克等列高 **或** 橫向膠卷；貓狗直幅多可試 `portrait.html` 交錯 |
| 人物（少） | `portrait.html` | 直幅 4:5 交錯，張數少也好看 |
| 商品／活動 | **先不上** 或 index 隱藏該區 | 避免訪客以為你是商業活動攝影 |

`photo/index.html` 總覽順序建議改為（日後改版時）：

1. 景點／風景  
2. 貓狗／寵物  
3. 人物  
4. （選填）生活隨拍

## 六、上站前每張檢查（簡表）

| 項目 | 建議 |
|------|------|
| 格式 | 上網用 **JPG**，sRGB；原始 RAW 不進 git |
| 長邊 | 約 **2000–2400 px**（風景可略大；人物／貓狗 1600–2000 即可） |
| 檔名 | `地點或主題_序號.jpg`，避免只有 `DSC_xxxx`（方便 alt 與搜尋） |
| 張數 | 景點每地 **12–24 張** 精選；貓狗每系列 **8–16 張**；人物 **6–12 張** 總量即可 |
| 重複 | 同構圖連拍只留 1–2 張 |

## 七、之後改版 Photo 區時（給自己的 checklist）

- [ ] `index.html` 三大區順序：景點 → 貓狗 → 人物  
- [ ] 重新命名標題（例：風景／街景 → **景點與風景**；新增 **貓狗** 頁或併入專頁）  
- [ ] 更新 `z_misc/site-guide/index.html` Photo 子項  
- [ ] 無商品／活動案時，從總覽拿掉或標「籌備中」  
- [ ] Nav 仍為「作品建置中」直到你滿意再上線

## 八、一句話總結

**平常整理：places / pets / people 三個資料夾；上站以景點為主、貓狗次之、人物精選少量；現成的 landscape／portrait 頁先借來用，商品與活動區先別當主分類。**
