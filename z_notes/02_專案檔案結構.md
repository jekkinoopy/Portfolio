# 專案檔案結構（依導覽列）

> 對應頂欄：`About · Graphic · Illustration`｜Logo｜`Photo · UI/UX · Web`  
> 互動版站圖（可點全站連結）：`z_misc/site-guide/`  
> 路徑均相對 repo 根目錄。

## 分類規則（最重要）

| 類型 | 放哪 | 判斷 |
|------|------|------|
| **Nav 頁** | repo 根目錄各區 | 頂欄有出現的 |
| **非 Nav** | `z_misc/` | 頂欄**沒有**的 HTML 全放這 |
| **共用資源** | `z_assets/` | css · js · images，各頁用 `../` 引用 |

### Nav 每一項 → 實際檔案（對照表）

頂欄每一格都對到下面其中一列。**不是**「只有兩個 html 是 Nav，其他資料夾不是」。

| 頂欄 | 實體路徑 | 說明 |
|------|----------|------|
| Logo／首頁 | `index.html` | 網站根目錄 `/`，所以是**單檔**放在 repo 根 |
| About | `about/index.html` | 一區一資料夾，入口叫 `index.html` |
| Graphic | `graphic.html` | 總覽頁，**刻意維持單檔**在根目錄（未整理） |
| Graphic 子頁 | `graphic/logoPkg.html` 等 | 子頁才在 `graphic/` 資料夾裡 |
| Illustration | `illustration/index.html` | 同上，資料夾 + index |
| Photo | `photo/index.html` + 子頁 | 同上 |
| UI/UX | `uiux/index.html` + 子頁 | 同上 |
| Web | `web/index.html` | 同上 |

**為什麼看起來怪？**  
歷史上 Graphic 沒動：`graphic.html` 留根目錄，子頁在 `graphic/`。其餘 Nav 後來改成「`資料夾/index.html`」。首頁 `index.html` 本來就必須在根目錄。所以側欄會看到 **2 個 .html 檔 + 6 個 Nav 資料夾**，其實**全是 Nav**，只是放法不統一。

**`z_` 開頭的才不是 Nav**：資源、報價、履歷、筆記等。

Graphic 是例外：總覽在根目錄 `graphic.html`，子頁在 `graphic/`（**未再整理**）。

### 側欄視覺分組（`z_` 前綴）

非 Nav 資料夾加 **`z_`**，字母排序時會排在 Nav 後面，側欄一眼分兩段：

```
about/ · graphic/ · illustration/ · photo/ · uiux/ · web/   ← Nav（資料夾，裡面是 index 或子頁）
index.html · graphic.html                                    ← 也是 Nav（首頁 + Graphic 總覽，單檔在根）
z_assets/ · z_misc/ · z_notes/ · z_原檔/                    ← 非 Nav
```

| 資料夾 | 內容 |
|--------|------|
| `z_assets/` | css · js · images |
| `z_misc/` | site-guide · price · resume · yumi · vedio |
| `z_notes/` | `00_` `01_` `02_` 三份 md 筆記 |
| `z_原檔/` | 原始素材備份 |
